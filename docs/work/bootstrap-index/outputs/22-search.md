## Changes

Wrote three files under `autopilot-support/index/features/search/`:

- `map.md` ‚Äî every CSV basename for `Search Functionality` listed with a one-line role, grouped by layer: GraphQL (`search_result_type.rb`), Selector (`search_results.rb`), Database (`enable_pgtrgm_extension.rb`, `add_gin_index_to_song_name.rb`), Tests (`search_spec.rb`). Opens each entry with non-obvious facts: `SearchResultType.resolve_type` dispatches `Song`‚Üí`SongType` and `OpenStruct`‚Üí`YoutubeResultType` (the `OpenStruct` branch is load-bearing because `YoutubeClient#search` returns `OpenStruct`, not models); the selector does local-first ‚Üí YouTube-fallback with library-exclusion via subquery; the selector currently uses plain ILIKE on name and does **not** invoke `Song.search`'s 3-tier scope; the GIN trigram index on `songs.name` co-exists with the newer text_search infrastructure rather than being replaced by it. "See also" calls out the deferral to `features/songs/` for `Song.search` scopes, the `text_search` tsvector column, and the multi-column trigram migration.

- `patterns.md` ‚Äî eleven pattern sections: (1) search is Postgres-native ‚Äî no Elastic/Algolia/Meili exists or should exist; (2) the 3-tier ranking on `Song.search` (fulltext via tsvector @@ plainto_tsquery, fuzzy via `<%` word-similarity, ILIKE substring fallback) ‚Äî tiers are recall safety nets, not re-ranks, and the CASE-shaped ORDER BY is intentional; (3) the `text_search` generated column is weighted A/A/B/C on `name`/`channel_title`/`youtube_tags`/`description` and you can't `ADD COLUMN` to a generated column (must drop/recreate); (4) the `immutable_array_to_string` SQL function exists only so `youtube_tags` can appear inside a generated column / functional index ‚Äî `array_to_string` is `STABLE` not `IMMUTABLE`, hence the wrapper; (5) two trigram indexes (GIN on `songs.name`, GiST on combined expression) for two different operators ‚Äî GIN can't order by `<<->`; (6) **the selector currently bypasses the 3-tier scope** (plain ILIKE) and the `search_spec` is written to ILIKE semantics ‚Äî flagged as both pattern and caveat for future wiring; (7) library-exclusion subquery uses `NOT IN (SELECT)` not a `LEFT JOIN` to avoid row multiplication; (8) local-first/YouTube-fallback is product-driven ‚Äî results are never blended, which is the *reason* `SearchResultType` is a union; (9) the `OpenStruct` resolve_type branch is fragile and load-bearing; (10) `lookahead` is plumbed but unused ‚Äî it's a future N+1 prevention hook; (11) blank-query short-circuits in `QueryType#search` and `Song.search` independently to prevent both DB hits and YouTube quota burn.

- `boundaries.md` ‚Äî owns the selector, the union type, the GIN trigram index on `songs.name`, the `pg_trgm` extension enablement, and the request-level specs. Ends at: `Song`'s search scopes / `text_search` / `immutable_array_to_string` / multi-column trigram index (all `features/songs/`); `YoutubeClient` and the `OpenStruct` shape (`features/youtube/`); `LibraryRecord` for the exclusion subquery (`features/music-library/`); the `search` field declaration (`structures/resources.md`); tag matching (subsumed by `youtube_tags` text array, not `Tag` model ‚Äî `features/tagging/`). Extension points: new searchable entities (selector branch + union `possible_types` + `resolve_type` + spec; new entities get their own model-level `search` scope rather than centralized ranking); new tier on `Song.search` (model + spec + maybe index); wire the selector to use `Song.search` (with `search_spec` re-confirmation); pagination via connection_type (avoid OFFSET); plumb `lookahead`. Do-not-build: no Elastic/OpenSearch/Meili/Algolia/Typesense (Postgres path is intentional); no searching `Tag`/`LibraryRecord` directly (extend `text_search` expression instead); no blending local + YouTube results; no raw `OpenStruct` from new selector paths without updating `resolve_type`; no calling `Song.search` outside this feature (too expensive); no `OFFSET` pagination on trigram queries. Schema invariants: `pg_trgm` must remain enabled, `immutable_array_to_string` must remain `IMMUTABLE`, `text_search` is `GENERATED ALWAYS AS ‚Ä¶ STORED` and cannot be assigned, `setweight` order encodes priority.
